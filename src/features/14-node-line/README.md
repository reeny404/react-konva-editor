## 기능 개요

`14-node-line`은 여러 개의 Rect 노드를 순서대로 연결하는 Connector Line 기능을 실험하는 화면입니다.

- 좌측 메인 캔버스에서 노드와 커넥터를 확인할 수 있습니다.
- 하단 우측의 `Create Connector` 패널에서 새 커넥터를 생성할 수 있습니다.
- 앱 전체 우측 패널(`AppLayout`의 `Properties`)에서는 현재 선택한 커넥터의 상세 속성을 편집할 수 있습니다.
- 상단 토글(`Locked`, `Selectable`, `Visible`)은 현재 존재하는 모든 커넥터에 일괄 적용됩니다.
- `Reset` 버튼을 누르면 초기 데모 상태로 되돌아갑니다.

## 빠른 테스트 방법

### 1. 초기 상태 확인

1. `14-node-line` 화면으로 이동합니다.
2. `Reset` 버튼을 눌러 초기 상태를 맞춥니다.
3. 기본으로 배치된 4개의 Rect 노드(Node1, Node2, Node3, Node4)가 보이는지 확인합니다.

초기 데모에는 커넥터가 자동 생성되어 있지 않기 때문에, 아래 단계에서 직접 커넥터를 만들어 테스트하면 됩니다.

### 2. 커넥터 생성 테스트

1. `Create Connector` 패널에서 `Connect Nodes` 항목에서 연결할 노드들을 2개 이상 선택합니다.
2. `Name`, `Line Style`, `Stroke`, `Stroke Width`, `Dash`, `Tension`, `Tooltip` 값을 입력합니다.
3. `Add Connector`를 눌러 커넥터를 생성합니다.
4. 생성 직후 해당 커넥터가 선택되고, 우측 `Properties` 패널에서 해당 커넥터 속성을 확인할 수 있습니다.

추천 테스트:

- `Node1 -> Node2`로 `straight`
- `Node1 -> Node2 -> Node4`로 `orthogonal`
- `Node1 -> Node2` 또는 `Node1 -> Node2 -> Node3`로 `curved`

### 3. 선택과 속성 편집 테스트

1. 캔버스에서 커넥터를 클릭합니다.
2. 우측 `Properties` 패널에 선택한 커넥터의 속성이 표시되는지 확인합니다.
3. `Name`, `Node IDs`, `Line Style`, `Stroke Width`, `Stroke`, `Dash`, `Tension`, `Tooltip`을 수정합니다.
4. 수정 내용이 캔버스에 반영되는지 확인합니다.

### 4. 상단 토글 테스트

1. `Visible`을 끄면 모든 커넥터가 한 번에 숨겨지는지 확인합니다.
2. `Visible`을 다시 켜면 모든 커넥터가 다시 보이는지 확인합니다.
3. `Selectable`을 끄면:
   - 모든 커넥터가 선택 불가능해지는지
   - 이미 선택되어 있던 커넥터도 즉시 선택 해제되는지
   - Rect 노드는 여전히 선택 가능한지
4. `Selectable`을 다시 켜면 커넥터를 다시 각각 클릭 선택할 수 있는지 확인합니다.
5. `Locked`를 켜면:
   - 각 커넥터는 여전히 선택 가능하지만
   - 우측 `Properties` 패널에서는 값이 보이되 수정은 불가능한지
6. `Locked`를 다시 끄면 편집이 다시 가능한지 확인합니다.

핵심 포인트:

- `Locked`, `Selectable`, `Visible`은 모두 "전체 커넥터 일괄 제어"입니다.
- 다만 `Locked`는 선택 자체를 막지 않습니다.
- 따라서 `Locked`가 켜져 있어도 개별 커넥터를 선택해서 속성값을 읽는 것은 가능합니다.

### 5. Reset 테스트

1. 커넥터를 여러 개 만든 뒤 속성을 바꿉니다.
2. 상단 `Reset` 버튼을 누릅니다.
3. 캔버스와 선택 상태가 초기 데모 상태로 돌아가는지 확인합니다.

## Create Connector 패널 설명

`Create Connector` 패널은 새 커넥터를 생성하는 입력 폼입니다.

### Name

- 커넥터 이름입니다.
- 생성 직후 캔버스 상단 표시와 우측 `Properties` 패널에서 사용됩니다.

### Connect Nodes

- 어떤 Rect 노드들을 어떤 순서로 연결할지 결정합니다.
- 선택 순서대로 `nodeIds` 배열이 만들어지고, 커넥터 경로도 그 순서를 따라 계산됩니다.
- 최소 2개 이상의 노드를 선택해야 커넥터를 생성할 수 있습니다.

### Line Style

커넥터 경로를 어떤 방식으로 계산할지 정합니다.

- `straight`
  - 각 노드의 중심점을 직선으로 바로 연결합니다.
  - 꺾이는 보정 없이 가장 단순한 형태입니다.
- `orthogonal`
  - 각 구간을 직각 경로로 만듭니다.
  - 현재 구현은 두 점 사이의 `중간 x 좌표(midX)`를 기준으로 먼저 수평 이동하고, 그 다음 수직 이동한 뒤, 마지막으로 목적지까지 이동합니다.
  - 즉 한 구간은 `가로 -> 세로 -> 가로` 형태로 꺾입니다.
- `curved`
  - 점들을 부드럽게 통과하는 곡선 형태입니다.
  - 2개 노드만 연결할 때는 두 점 사이 중간 지점을 기준으로 위쪽에 보조 점 하나를 추가해서 활처럼 휘는 형태를 만듭니다.
  - 3개 이상 노드를 연결할 때는 각 노드 중심점을 사용하고, Konva의 `tension`으로 부드럽게 연결합니다.

## Orthogonal과 Curved의 차이

### Orthogonal

- 경로가 직선 세그먼트들의 조합입니다.
- 각 구간이 명확하게 꺾여 보입니다.
- 꺾이는 기준은 현재 구현상 "이전 점과 다음 점의 x축 중간 지점"입니다.
- 라우팅 충돌 회피나 장애물 회피는 하지 않고, 단순한 규칙 기반 경로입니다.

### Curved

- 같은 점들을 기준으로 하되, 직각 꺾임 대신 곡선으로 연결합니다.
- `tension` 값을 통해 얼마나 둥글게 휘는지 조절할 수 있습니다.
- 시각적으로 더 유연하고 흐르는 느낌을 줍니다.

## 개별 입력값 설명

### Stroke

- 선 색상입니다.

### Stroke Width

- 선 두께입니다.
- 선택된 커넥터는 화면에서 강조 표현을 위해 실제 값보다 조금 더 두껍게 보일 수 있습니다.

### Dash

- 점선 패턴입니다.
- 예: `12, 8`
- 의미:
  - 12px 선을 그리고
  - 8px 비우는 패턴을 반복합니다.

### Tension

- 곡선의 부드러움 정도를 조절합니다.
- `curved` 스타일에서 가장 의미가 크며, 값이 커질수록 더 둥글고 유연한 곡선 느낌이 납니다.
- `straight`나 `orthogonal`에서는 시각 효과가 상대적으로 제한적이거나 거의 의미가 없을 수 있습니다.

### Tooltip

- 커넥터에 마우스를 올렸을 때 보여줄 텍스트입니다.
- 현재 구현은 커넥터 경로의 중간쯤 되는 세그먼트를 찾은 뒤, 그 세그먼트의 중심 좌표에 툴팁을 배치합니다.
- 실제 렌더링은 `Canvas.tsx`의 `Label`, `Tag`, `Text` 조합으로 구성되어 있습니다.

## 우측 Properties 패널 설명

앱 전체 우측 패널은 현재 선택된 노드의 속성을 편집하는 공용 패널입니다.

이 기능에서 커넥터를 선택하면 아래 속성을 편집할 수 있습니다.

- `Name`
- `Node IDs`
- `Line Style`
- `Stroke Width`
- `Stroke`
- `Dash`
- `Tension`
- `Tooltip`

설명:

- `Node IDs`는 쉼표로 구분된 노드 id 목록입니다.
- 값을 바꾸면 커넥터가 어떤 노드들을 어떤 순서로 잇는지도 함께 바뀝니다.
- 커넥터가 `Locked` 상태이면 값은 그대로 표시되지만 편집은 막힙니다.

## 상단 토글 설명

### Locked

- 모든 커넥터를 한 번에 읽기 전용 상태로 전환합니다.
- 선택은 가능하지만, 우측 `Properties` 패널에서 속성 편집은 막힙니다.
- 즉, "선택 금지"가 아니라 "편집 금지"입니다.

### Selectable

- 모든 커넥터의 선택 가능 여부를 한 번에 제어합니다.
- 끄면 커넥터를 클릭해도 선택되지 않습니다.
- 끄는 순간 이미 선택 중이던 커넥터도 자동으로 선택 해제됩니다.

### Visible

- 모든 커넥터의 표시 여부를 한 번에 제어합니다.
- 끄면 커넥터가 전부 숨겨집니다.
- 다시 켜면 숨겨진 커넥터가 다시 나타납니다.

## 구현 메모

### 경로 계산 기준

- `straight`: 각 Rect 중심점들을 그대로 연결
- `orthogonal`: 인접한 두 중심점 사이마다 `midX`를 만들어 직각 경로 구성
- `curved`:
  - 2개 점일 때는 가운데 위쪽 보조점을 추가
  - 3개 이상일 때는 중심점 배열 자체를 사용하고 `tension`으로 곡률 형성

### 툴팁 위치 계산

- 전체 포인트 목록에서 가운데에 가까운 세그먼트를 찾습니다.
- 해당 세그먼트의 시작점과 끝점의 중간 좌표를 툴팁 기준점으로 사용합니다.
