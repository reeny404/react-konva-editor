### 기능 요약

- 캔버스 내 마우스 드래그를 통한 새로운 사각형 도형(Rect) 생성
- 부모-자식 트리 관계를 활용한 다중 객체 그룹화
- Konva `Group` 컴포넌트를 이용한 자식 노드의 상대 좌표 렌더링 및 동시 이동

### 조작법

- **도형 그리기**: 캔버스 빈 공간(배경)을 클릭하고 마우스를 드래그하여 영역을 지정하면 새로운 사각형(`Rectangle 1`, `Rectangle 2` 등)이 생성됩니다.
- **도형 그룹핑 (부모-자식 연결)**:
  - (현재 구현 스펙 기준) 직전에 생성된 도형이 만약 있다면, 이후 새로 그려지는 도형은 직전에 그려진 도형의 "자식(Child)"으로 자동 할당됩니다.
  - 마우스 드래그를 완료할 때, 새로운 자식 도형의 `x`, `y` 좌표는 캔버스의 절대 좌표가 아닌 **부모 도형 기준 상대 좌표**로 계산되어 들어갑니다.
- **그룹 이동**:
  - 생성된 사각형 무리 중 **가장 바깥쪽 부모**를 드래그하면, `<Group>`으로 묶여있는 자식 사각형들도 함께 부모를 따라 이동합니다.

### 구현 포인트

#### 1. Flat 데이터를 Tree 형태로 렌더링

`useDocumentStore`에는 평면적인 1차원 데이터 배열(Flat list) 형태로 도형 데이터가 저장되어 있습니다 (단순히 `parentId` 필드로 관계 명시).
렌더링 단계(Canvas.tsx)에서 `parentId`가 없는 `rootNodes`들을 먼저 찾고, 이들 내부에 재귀 함수(`renderNode`)를 호출하여 자기 자신을 부모로 두는 자식 도형들을 `<Group>` 컴포넌트 하위에 차례차례 렌더링합니다.

#### 2. 물리적 그룹 동기화 (Konva Group 처리)

기존 로직에서는 부모가 이동할 때 자식들의 좌표(`deltas`)를 수동으로 계산해 `documentCommands.patchNode`로 모두 업데이트해줘야 하는 비효율이 있었습니다.
이를 코어 렌더링 단에서 `<Group>`을 쓰도록 리팩토링함으로써 다음과 같은 변화가 생겼습니다.

- 부모 도형이 드래그될 때, 자식 좌표까지 패치할 필요 없이 **오직 부모의 좌표만 패치**하면 동작합니다.
- 자식 도형들의 변위 동기화를 Konva 엔진 레벨로 이관하여 속도와 코드 효율성을 대폭 끌어올렸습니다.

#### 3. 클릭 및 이벤트 전파(Bubbling) 제한

자식 객체를 선택(클릭)하거나 드래그(`onDragStart`)할 때, 부모 그룹까지 동시에 이벤트가 전파되어 부모가 함께 선택되버리는 현상을 방지하고자 `e.cancelBubble = true` 를 적극적으로 활용하여 타겟 도형만 정확히 선택되도록 격리시켰습니다.

### 파일 구성

- `Canvas.tsx`: 마우스 이벤트를 컨트롤하여 Temp Rect를 띄우고, 노드 구조를 재귀(`renderNode`) 순차적으로 펼쳐서 렌더링합니다.
- `hooks/useDrawing.ts`: 드래그로 도형 크기 지정 및 마우스 Up 순간 store에 Rect 추가, **자식 상대 좌표 계산 로직**을 담당합니다.
- `commands/documentCommands.ts`: 노드 수정(Patch) 시 기존에 수동으로 수행하던 '자식 위치 동기화' 로직이 제거되어 순수 노드 업데이트와 History Redux 관리만을 수행합니다.
