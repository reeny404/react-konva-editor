### 기능 요약

- **점 기반 폴리곤 그리기**: 마우스 클릭으로 꼭짓점을 추가하고, 시작점을 다시 클릭하여 도형 완성
- **사각형 기반 폴리곤 그리기**: 드래그로 사각형 영역을 지정하면 자동으로 4개의 꼭짓점을 가진 폴리곤으로 변환
- **정교한 꼭짓점 편집**:
  - 완성된 폴리곤 더블 클릭 시 편집 모드 진입
  - 꼭짓점 드래그로 위치 수정
  - 변(Edge) 클릭 시 새로운 꼭짓점 추가
  - 꼭짓점 우클릭 시 해당 점 삭제 (최소 3개 유지)
- **v2 아키텍처 원칙 준수**: 데이터 가공(Adapter), 상태 변경(Command), 렌더링(Canvas)의 엄격한 분리

### 조작법

- **툴 선택**: 상단 버튼을 통해 '점 기반' 또는 '사각형 기반' 모드 선택
- **그리기 (점)**: 캔버스 클릭 시 점 추가 -> 시작점(녹색/적색 표시) 클릭 시 완료
- **그리기 (사각형)**: 마우스 드래그로 영역 지정 -> 마우스 떼면 완료
- **편집 모드 진입**: 완성된 폴리곤 **더블 클릭**
- **꼭짓점 이동**: 편집 모드에서 흰색 원(꼭짓점) 드래그
- **꼭짓점 추가**: 편집 모드에서 폴리곤의 변(선분) 클릭
- **꼭짓점 삭제**: 편집 모드에서 꼭짓점 **우클릭**
- **편집 종료**: 폴리곤 외부(배경) 클릭

### 구현 포인트 (Architecture v2)

#### 1. Adapter 패턴 (`usePolygonAdapter.ts`)

- **역할**: Store의 로우 데이터(DB 모델)를 Canvas가 바로 그릴 수 있는 **ViewModel**로 가공합니다.
- **특징**:
  - `documentStore`(데이터)와 `layerStore`(구조)를 조합하여 현재 레이어 순서에 맞는 폴리곤 목록을 생성합니다.
  - 좌표 평탄화(`flattenPoints`), 에지 계산(`getPolygonEdges`), 선택 및 편집 상태 판별 로직을 캡슐화합니다.
  - Canvas는 이 훅이 제공하는 완성된 데이터만 사용하여 **순수 렌더링**만 수행합니다.

#### 2. Command 패턴 (`polygonCommands.ts`)

- **역할**: 모든 상태 변경(점 추가, 편집 시작, 노드 생성 등)을 전담합니다.
- **특징**:
  - `layerCommands`와 연동되어 노드 추가/수정 시 **Undo/Redo** 성능을 보장합니다.
  - 비즈니스 로직(최소 점 개수 체크, 이름 자동 생성 등)을 Store나 UI에서 분리하여 관리합니다.

#### 3. Layer-Document Store 연동

- 신규 폴리곤 생성 시 `documentStore`에 실제 데이터를 저장하고, 동시에 `layerStore`의 `layerMapper`를 통해 현재 활성화된 레이어에 소속시킵니다.
- 만약 활성 레이어가 없다면 최상위 레이어에 자동으로 할당합니다.

### 파일 구성

- `index.tsx`: 가공된 ViewModel을 받아 화면에 그리는 메인 렌더러
- `hooks/usePolygonAdapter.ts`: 데이터를 UI 모델로 변환하는 어댑터 (ViewModel 추출)
- `commands/polygonCommands.ts`: 상태 변경 및 비즈니스 로직 실행 (Single Write Path)
- `stores/polygonToolStore.ts`: 그리기 모드, 임시 드래프트 등 도구 전용 상태 관리
- `utils/polygonUtils.ts`: 수학적 계산(Bounding Box, 좌표 필터링 등) 유틸리티
- `types/polygon.ts`: 폴리곤 피처 전용 타입 정의
