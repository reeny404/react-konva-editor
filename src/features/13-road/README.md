### 기능 요약

- **점 기반 도로(Line) 그리기**: 마우스 클릭으로 점을 추가하고, 더블 클릭으로 완성하여 자유로운 경로의 도로를 생성합니다.
- **실시간 도로 폭 조절**: 우측 패널의 `Stroke Width`를 통해 도로의 두께를 실시간으로 편집할 수 있습니다.
- **2중 레이어 렌더링**:
  - **도로 몸체**: 유저가 설정한 색상과 두께를 반영하는 실제 도로 바닥면.
  - **선택/중앙선**: 선택 시에만 나타나는 고정된 두께(6px)의 파란색 가이드 라인으로, 시인성을 높였습니다.
- **정교한 꼭짓점 편집**:
  - 완성된 도로 **더블 클릭** 시 편집 모드 진입.
  - 꼭짓점 드래그로 위치 수정 및 선분(Edge) 클릭 시 새로운 점 삽입.
  - 꼭짓점 우클릭 시 해당 점 삭제 (최소 2개 유지).
- **v2 아키텍처 원칙 준수**: 데이터 가공(Adapter), 상태 변경(Command), 렌더링(Canvas)의 엄격한 분리.

### 조작법

- **그리기 시작**: 빈 공간 클릭 시 드로잉 모드 진입 -> 점 추가 -> **더블 클릭** 시 최종 완성.
- **정밀 클릭(Selection)**: 얇은 도로도 쉽게 선택할 수 있도록 실제 두께와 상관없이 최소 16px 이상의 마우스 감도(`hitStrokeWidth`)를 보장합니다.
- **편집 모드 진입**: 이미 그려진 도로를 **더블 클릭**.
- **꼭짓점 이동**: 편집 모드에서 나타나는 흰색 원(꼭짓점)을 드래그.
- **꼭짓점 추가**: 편집 모드에서 도로의 선분(변)을 클릭.
- **꼭짓점 삭제**: 편집 모드에서 특정 꼭짓점 위에서 **우클릭**.
- **편집 종료**: 도로 외부(배경) 클릭.

### 구현 포인트 (Architecture v2)

#### 1. Adapter 패턴 (`useRoadAdapter.ts`)

- **역할**: `documentStore`에 저장된 로우 데이터를 컴포넌트가 렌더링하기 편한 **ViewModel**로 가공합니다.
- **특징**:
  - 좌표 배열 평탄화(`flattenedPoints`) 및 편집 가능한 선분 데이터(`edges`) 생성을 캡슐화합니다.
  - `documentNodes`와 `selectedIds`를 직접 구독하여, 데이터 변경 시 즉각적으로 UI에 반영되도록 리렌더링을 유도합니다.

#### 2. Command 패턴 (`roadCommands.ts`)

- **역할**: 도로 생성, 편집, 점 삽입/삭제 등 모든 상태 변경 비즈니스 로직을 전담합니다.
- **특징**:
  - `layerCommands`와의 연동을 통해 노드 추가 및 속성 수정 시 **Undo/Redo** 성능이 보장됩니다.
  - 더블 클릭 시 발생할 수 있는 중복 좌표 삽입 등을 방지하는 로직이 내장되어 있습니다.

#### 3. UX 최적화 (2-Layer Rendering)

- **문제**: 도로 폭(Stroke)을 데이터와 직접 연결하면, 선택되었을 때 강조 색상을 입히는 과정에서 실제 데이터인 도로 폭이 가려지는 문제가 있었습니다.
- **해결**: 실제 데이터를 그리는 '몸체 레이어'와 강조 효과를 보여주는 '선택 레이어'를 분리하여, 편집 중에도 실제 도로 두께의 변화를 실시간으로 관찰할 수 있습니다.

### 파일 구성

- `index.tsx`: 가공된 ViewModel을 받아 화면에 그리는 메인 렌더러
- `hooks/useRoadAdapter.ts`: 상태 데이터를 UI 모델로 변환하는 어댑터
- `commands/roadCommands.ts`: 상태 변경 및 비즈니스 로직 실행 (Undo/Redo 대응)
- `stores/roadToolStore.ts`: 그리기 모드, 임시 드래프트 등 도구 전용 상태 관리
- `utils/roadUtils.ts`: 수학적 계산(Bounding Box, 좌표 변환 등) 유틸리티
- `types/road.ts`: 도로 피처 전용 타입 정의
